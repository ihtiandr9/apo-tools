# Спецификация формата RKA (Apogey tape format)

Формат файла для хранения программ BK-0010 на кассетном магнитофоне (Apogey).

## Форматы RKA файла

Существует два основных формата RKA файлов:

### 1. Именованный BASIC (Named BASIC)

Используется для программ на BASIC с именем файла.

```
+----------+
|  Адрес загрузки 2 байта = 0xE6E6 (маркер)     |
|  Адрес конца 2 байта = 0xE6E6 (маркер)         |
|  Имя файла (ASCIIZ)                             |
|  Паддинг 512 байт (нули)                       |
|  Синхробайт 1 байт = 0xE6                      |
|  2 байта служебной информации                   |
+----------+
|  Данные программы (тело)                        |
|  Кодировка: символы Apogey (KOI7)              |
+----------+
```

- Адрес загрузки и адрес конца `0xE6E6` — специальные маркеры для именованного BASIC
- Порядок байтов: **старший байт, затем младший** (big-endian)
- Паддинг фиксирован: **512 байт** после имени файла
- После паддинга: **синхробайт `0xE6` + 2 байта служебной информации**
- При распаковке имя файла выводится в stderr: `Filename: <имя>`
- В `test.rka` (смещение 0x00-0x03): `E6 E6 E6 E6`

### 2. Двоичная программа (Binary Program)

Используется для машинного кода без имени файла.

```
+----------+
|  Адрес загрузки 2 байта (старший, младший)      |
|  Адрес конца 2 байта (старший, младший)          |
+----------+
|  Данные программы (тело)                        |
|  Кодировка: символы Apogey (KOI7)              |
+----------+
```

- Адрес загрузки и адрес конца — **реальные адреса** в памяти BK-0010
- Порядок байтов: **старший байт, затем младший** (big-endian)
- В `bin_program.rka` (смещение 0x00-0x03): `11 00 11 E3` → загрузка=`0x1100`, конец=`0x11E3`
- Размер данных = адрес конца - адрес загрузки + 1 = `0x11E3 - 0x1100 + 1 = 0x00E4` (228 байт)
- Имя файла **отсутствует**
- **Паддинг отсутствует**
- **Служебные байты отсутствуют** — данные начинаются сразу после адреса конца (смещение 0x04)
- Фактический размер файла `bin_program.rka` = 237 байт (4 байта заголовок + 228 байт кода + 5 байт в конце)
- **Распаковка не реализована** — выводится сообщение об ошибке

## Детальный разбор (общий для обоих форматов)

### 1. Адрес загрузки (2 байта, старший + младший)

- 2-байтовый адрес загрузки программы в памяти BK-0010
- Порядок байтов: **старший байт, затем младший** (big-endian)
- Для именованного BASIC: `0xE6E6` (маркер)
- Для двоичной программы: реальный адрес (например, `0x0000`-`0xFFFF`)
- В `test.rka` (смещение 0x00-0x01): `E6 E6`
- В `bin_program.rka` (смещение 0x00-0x01): `11 00` → адрес `0x1100`

### 2. Адрес конца (2 байта, старший + младший)

**Для именованного BASIC:**
- Маркер `0xE6E6`

**Для двоичной программы:**
- Адрес конца (end address) — последний адрес программы в памяти
- Порядок байтов: **старший байт, затем младший** (big-endian)
- Размер данных = адрес конца - адрес загрузки + 1
- В `bin_program.rka` (смещение 0x02-0x03): `11 E3` → конец `0x11E3`
- Размер = `0x11E3 - 0x1100 + 1 = 0x00E4` (228 байт кода)

### 3. Имя файла (только для именованного BASIC)

- Сразу после заголовка (4 байта) идёт ASCIIZ-строка (имя программы)
- При распаковке выводится в stderr: `Filename: <имя>`
- Для двоичной программы — **отсутствует**
- В `test.rka`: `SIMPLEBASIC` + `0x00` (смещение 0x04)
- Максимальная длина не ограничена явно

### 4. Пропуск пустого пространства (паддинг)

**Только для именованного BASIC:**
- Паддинг фиксирован: **512 байт** нулей (`0x00`) после имени файла
- В `test.rka` паддинг занимает 512 байт (до смещения `0x210`)
- После паддинга: **синхробайт `0xE6` + 2 байта служебной информации**
- Затем идут данные программы

**Для двоичной программы:**
- Паддинг **отсутствует**
- Служебные байты **отсутствуют**
- Данные начинаются сразу после адреса конца (смещение 0x04)

### 5. Служебные байты (3 байта, только для Named BASIC)

- После паддинга читаются 3 байта, которые отбрасываются
- В `test.rka` (смещение `0x210`): `E6 CD E3`
- Назначение не задокументировано в коде

### 6. Тело программы

- Данные в кодировке символов Apogey (KOI7-N1)
- Обрабатываются unpack-ом по одному байту:
  - `0x0D` (CR) → преобразуется в `0x0A` (LF)
  - Управляющие символы `< 0x20` (кроме `0x0D`) → заменяются на `0x0A`
  - Остальные → табличное преобразование в Unicode

### Примечание по коду unpack()

В текущей реализации `edb.c:unpack()` (строки 73-150):
- Читает заголовок (4 байта): `load_addr` и `end_addr` (big-endian)
- Определяет тип файла: если оба маркера `0xE6E6` → Named BASIC, иначе → Binary
- Для Named BASIC:
  - Читает и выводит имя файла в stderr
  - Пропускает 512 байт паддинга
  - Пропускает 3 служебных байта
  - Декодирует тело
- Для Binary program: выводит сообщение об ошибке (заглушка)

## Таблица символов Apogey (KOI7)

Формат RKA использует кодировку KOI7 (КОИ-7) — советский стандарт 7-битного кодирования символов.

### Основная таблица (определена в `edb.c:getUnicodeSymbol()`, строки 17-29)

Таблица `c_apogeySymbols` — 128 символов (индекс = байт в RKA файле):

| Байт | Символы |
|------|---------|
| 0x00-0x1F | ` ▘▝▀▗▚▐▜ ★⬯↑⬮ ↣↓▖▌▞▛▄▙▟█∼≈╋┃━↢✿▕` (спецсимволы Apogey) |
| 0x20-0x7E | Стандартный ASCII (печатные символы) |
| 0x7F | `▒` |
| 0x80-0x8F | `ЮАБЦДЕФГХИЙКЛМНО` (русские буквы, кодировка KOI7-N1) |
| 0x90-0x9F | `ПЯРСТУЖВЬЫЗШЭЩЧ▒` (русские буквы, кодировка KOI7-N1) |

### Соответствие KOI7-N1

KOI7-N1 (ГОСТ 13052-74) использует 7-битные коды для русских букв:
- `Ю` = 0x6A (106), `А` = 0x61 (97), `Б` = 0x62 (98), и т.д.
- В RKA формате для русских букв используется **8-битное расширение**: старший бит установлен (0x80+)

### Таблица обратного преобразования (`charCodeAt()`, строки 31-51)

Используется при pack (кодировании ASM → RKA):

| Байт | Символы |
|------|---------|
| 0x00-0x1F | ` ▘▝▀▗▚▐▜ ★⬯↑⬮ ↣↓▖▌▞▛▄▙▟█∼≈╋┃━↢✿▕` |
| 0x20-0x7E | Стандартный ASCII |
| 0x80-0x8F | `abwgdevzijklmnop` (транслит для 0x80-0x8F) |
| 0x90-0x9F | `rstufhc~{}.yx|`q` (транслит для 0x90-0x9F) |

**Примечание**: При декодировании русских букв (Unicode > 0x400) в `unpack()` добавляется UTF-8 префикс `0xD0` для корректного вывода в UTF-8 терминал.

## Особенности pack/unpack

### Unpack (`rkaexchange unpack <tape> <asm>`)

- Читает RKA, пропускает заголовок, декодирует тело
- Для русских букв (Unicode > 0x400) добавляет UTF-8 префикс `0xD0`
- Код: `edb.c:unpack()` (строки 73-135)

### Pack (`rkaexchange pack <asm> <tape>`)

- Кодирует текст через `charCodeAt()` (обратная таблица)
- Пропускает UTF-8 префикс `0xD0` при чтении входного файла
- **Внимание**: согласно коду `pack()` (строки 137-157) НЕ добавляет RKA-заголовок (маркер, имя файла) — функционал формирования заголовка при упаковке может быть не завершён
- Код: `edb.c:pack()` (строки 137-157), `edb.c:encode()` (строки 54-71)

## Статус реализации

- **Unpack**: работает корректно, читает существующие RKA-файлы
- **Pack**: только кодирование символов, без формирования заголовка (потенциальный баг)

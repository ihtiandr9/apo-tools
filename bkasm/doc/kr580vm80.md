# КР580ВМ80 — краткая спецификация

Советский клон Intel 8080A. 8-битный процессор, 16-битная шина адреса (64 КБ адресного пространства).

## Регистры

| Регистр | Разрядность | Назначение |
|---------|-------------|------------|
| A | 8 | Аккумулятор |
| B, C, D, E, H, L | 8 | Общего назначения |
| SP | 16 | Указатель стека |
| PC | 16 | Счётчик команд |
| PSW (AF) | 16 | Флаги (мл. байт) + A (ст. байт), для PUSH/POP |

Регистры B,C,D,E,H,L спариваются: BC, DE, HL.
M — обращение к памяти по адресу из HL.

**Флаги:** S (знак), Z (ноль), AC (полуперенос), P (чётность), C (перенос).

## Формат машинной команды

| Размер | Байт 1 | Байт 2 | Байт 3 |
|--------|--------|--------|--------|
| 1 байт | opcode | — | — |
| 2 байта | opcode | data/port | — |
| 3 байта | opcode | addr_lo | addr_hi |

Адреса/16-битные данные — little-endian (младший байт первым).

Кодировка многих инструкций следует шаблону Intel 8080:
- `MOV r1,r2` — `01 DDD SSS` (DDD — приёмник, SSS — источник)
- `ADD/ADC/SUB/SBB/ANA/ORA/XRA/CMP r` — `10 XXX SSS` (XXX — код операции, SSS — регистр)
- `INR/DCR r` — `00 XXX SSS` (XXX = 100/101)
- `MVI r` — `00 XXX 110`, следом байт данных
- Условные переходы/вызовы: биты 3,5 — код условия (NZ, Z, NC, C, PO, PE, P, M)

### 1-байтовые (без операндов)

|-----------|--------|----------|
| NOP | 0x00 | Нет операции |

| RIM | 0x20 | Чтение маски прерываний |
| SIM | 0x30 | Установка маски прерываний |
| STC | 0x37 | Установка флага C |

| CMA | 0x2F | Инверсия A |
| DAA | 0x27 | Десятичная коррекция A |
| RLC | 0x07 | Циклический сдвиг A влево |
| RRC | 0x0F | Циклический сдвиг A вправо |
| RAL | 0x17 | Сдвиг A влево через C |

| RET | 0xC9 | Возврат из подпрограммы |

### 1-байтовые (с регистром)

**MOV r1, r2** — `01 DDD SSS`

| Мнемоника | Код | DDD/SSS |
|-----------|-----|---------|
| B | 000 | |
| C | 001 | |
| D | 010 | |
| E | 011 | |
| H | 100 | |
| L | 101 | |
| M | 110 | (память по HL) |
| A | 111 | |

Пример: `MOV A,B` = 0x78 (01 111 000)

**INR r** — `00 100 SSS` (базовый 0x04), **DCR r** — `00 101 SSS` (базовый 0x05)

**ADD r** (0x80), **ADC r** (0x88), **SUB r** (0x90), **SBB r** (0x98),
**ANA r** (0xA0), **XRA r** (0xA8), **ORA r** (0xB0), **CMP r** (0xB8):
— `10 XXX SSS`, где XXX = код операции, SSS = регистр

**INX rp** — базовый 0x03, **DCX rp** — базовый 0x0B,
**DAD rp** — базовый 0x09, **LDAX rp** — базовый 0x0A,
**STAX rp** — базовый 0x02:
— регистровая пара: BC=00, DE=01, HL=10, SP=11
— биты 4-3 кодируют пару

**PUSH rp** — базовый 0xC5, **POP rp** — базовый 0xC1:
— пары: BC, DE, HL, PSW (AF)

### 2-байтовые (opcode + 1 байт данных)

| Мнемоника | Opcode | Операнд | Описание |
|-----------|--------|---------|----------|
| MVI r,byte | 0x06 + (r<<3) | 8 бит | Загрузка константы в регистр |
| ADI byte | 0xC6 | 8 бит | A += byte |
| ACI byte | 0xCE | 8 бит | A += byte + C |
| SUI byte | 0xD6 | 8 бит | A -= byte |
| SBI byte | 0xDE | 8 бит | A -= byte + C |
| ANI byte | 0xE6 | 8 бит | A &= byte |
| ORI byte | 0xF6 | 8 бит | A |= byte |
| CPI byte | 0xFE | 8 бит | A - byte (сравнение) |
| XRI byte | 0xEE | 8 бит | A ^= byte |
| OUT port | 0xD3 | 8 бит | Вывод на порт |
| IN port | 0xDB | 8 бит | Ввод с порта |
| RST n | 0xC7 + (n<<3) | — | Рестарт на адрес 8*n |

### 3-байтовые (opcode + 2 байта адреса/данных)

| Мнемоника | Opcode | Операнд | Описание |
|-----------|--------|---------|----------|
| JMP addr | 0xC3 | 16 бит | Безусловный переход |
| CALL addr | 0xCD | 16 бит | Вызов подпрограммы |
| LXI rp,word | 0x01 + (rp<<4) | 16 бит | rp = word |
| LDA addr | 0x3A | 16 бит | A = [addr] |
| STA addr | 0x32 | 16 бит | [addr] = A |
| LHLD addr | 0x2A | 16 бит | HL = [addr] |
| SHLD addr | 0x22 | 16 бит | [addr] = HL |

**Условные:** Jcc addr (C2/C3/D2/...), Ccc addr (C4/CD/D4/...)

| Условие | Флаги | Jcc opcode | Ccc opcode |
|---------|-------|------------|------------|
| NZ | Z=0 | 0xC2 | 0xC4 |
| Z | Z=1 | 0xCA | 0xCC |
| NC | C=0 | 0xD2 | 0xD4 |
| C | C=1 | 0xDA | 0xDC |
| PO | P=0 | 0xE2 | 0xE4 |
| PE | P=1 | 0xEA | 0xEC |
| P | S=0 | 0xF2 | 0xF4 |
| M | S=1 | 0xFA | 0xFC |

## Директивы ассемблера

| Директива | Описание |
|-----------|----------|
| ORG addr | Установка адреса размещения |
| DB byte | Байт данных |
| DW word | Слово данных (2 байта, little-endian) |
| END | Конец программы |
| EQU expr | Присвоение константы метке |
| label: | Определение метки (текущий адрес) |

## Формат вывода

Ассемблер генерирует бинарный файл `prog.bin` — машинный код без заголовков, начиная с адреса, указанного в `ORG`.
